## Overview

The `geb/` monorepo root currently presents two large generated files,
`README` (plain text, 125 KB) and `README.md` (markdown, 207 KB), both
emitted by the Common Lisp MGL-PAX documentation pipeline. They document
the original Common Lisp implementation, which is no longer the site of
active development. Active development is the Lean 4 formalisation under
`geb-lean/`, preceded by the Idris-2 implementation under `geb-idris/`.
The Lean and Idris material is largely undocumented at the index level,
and the obsolete Common Lisp manual occupies the project's front page.

This step restructures the documentation entry points so that the root
`README.md` becomes a hand-authored documentation index, the obsolete
Common Lisp material is relocated beneath `docs/` and marked historical,
and each subproject is reachable through its own documentation index. It
is the first of two phases: this phase establishes structure and
navigation only; a later phase authors new explanatory content.

## Goal

After this step:

- The root `README.md` is a hand-authored file: a short identity
  statement plus a categorized index linking to each subproject's
  documentation and to the relocated Common Lisp manual.
- All obsolete Common Lisp documentation material lives under
  `docs/common-lisp/`.
- The Common Lisp documentation generator, if rerun, writes to
  `docs/common-lisp/manual.md` rather than to the repository root, so a
  regeneration never overwrites the hand-authored root `README.md`.
- `geb-idris/docs/index.md` exists as a skeleton index of the existing
  Idris documentation files.
- `geb-lean/docs/index.md` is unchanged, reachable from the root index.

## Non-goals

- No new explanatory documentation is authored in any subproject. The
  only new prose is the root `README.md` identity statement and the
  navigational labels in the two index files.
- `geb-lean/docs/index.md` and its referenced files are not edited.
- Extending CI or pre-push markdownlint and doctoc coverage to
  parent-level (non-`geb-lean/`) markdown is out of scope; it is a
  separate concern recorded under Follow-ups.
- No `.lean` files are touched; the Lean pre-commit triad does not apply.

## Current state

- `README` and `README.md` at the repository root are generated by
  `mgl-pax:update-asdf-system-readmes` (see `docs/documentation.lisp`,
  function `build-docs`), invoked with `:formats '(:markdown :plain)`.
  That function writes both files to the ASDF system root; it has no
  target-directory parameter. The most recent non-generated change to
  `README.md` is the manual insertion of a `geb-lean` pointer near the
  top (commit `8693ed61`).
- `docs/` (repository root) holds the Common Lisp documentation sources
  `documentation.lisp`, `glossery.lisp`, `package.lisp`, and the
  maintainer document `maintainers-guide.org`. The ASDF system
  `geb/documentation` (in `geb.asd`) declares `:pathname "docs/"` with
  components `package`, `glossery`, `documentation`.
- `geb-idris/docs/` holds four markdown files and no index:
  `geb-syllabus.md`, `mldirichf-generalization.md`,
  `profunctor-end-characterization.md`,
  `twisted-arrow-copresheaf-analysis.md`. `geb-idris/` also has
  `README.md` and `EXAMPLES.md`.
- `geb-lean/docs/` holds an existing `index.md` plus many topic files.
- `geb-agda/` holds `README.md` and assorted artefacts; it is an
  original effort, not under active development.
- Parent-level CI markdownlint (`.github/workflows/markdown-lint.yml`)
  lints only `geb-lean/**/*.md`. The markdownlint configuration lives at
  `geb-lean/.markdownlint-cli2.jsonc`.

## Changes

Each numbered item is realized as one or more commits under the commit
granularity principle below. Items are ordered so that every commit
reads as a pure move, a pure deletion, or a pure skeleton creation.

Before moving the root `README`/`README.md`, confirm what links into
them so no other file's reference is silently broken. Verified inbound
references at spec time: no CI workflow, no `geb.asd` entry, and no
`src/` file links to the root README files; `geb-lean/README.md`,
`geb-idris/README.md`, and `geb-agda/README.md` link to `../README.md`
(which the new root index satisfies) rather than to the moving file's
new location. The plan repeats this check, including a grep of
`geb-idris/docs/*.md` and `geb-lean/docs/*.md` for root-README links,
immediately before the move.

1. Relocate the generated Common Lisp manual. Move the root `README.md`
   to `docs/common-lisp/manual.md`. This commit contains no other
   change, so version control records a pure rename. At this point the
   repository root has no `README.md`. The relocated file is frozen
   generator output: its body retains relative links written for the
   repository root, including a live link to `geb-lean/README.md`
   (the manually inserted pointer from commit `8693ed61`) that no longer
   resolves from `docs/common-lisp/`, alongside pre-existing dead links
   (`./tests/cover-index.html`, `./tests/report.html`). These stale
   links are accepted, not repaired: `manual.md` is not authored or
   linted here (see Markdownlint and doctoc), and a future regeneration
   (Change 4) emits correct links for the new location. Repairing them
   in this commit would forfeit the pure-rename property.
2. Relocate the Common Lisp documentation sources. Move
   `docs/documentation.lisp`, `docs/glossery.lisp` (sic; the filename
   is misspelled in the tree), `docs/package.lisp`, and
   `docs/maintainers-guide.org` into `docs/common-lisp/`. Update
   `geb.asd`: the `geb/documentation` system's `:pathname "docs/"`
   becomes `:pathname "docs/common-lisp/"`. The move and its single
   reference update form one conceptual operation. The three `.lisp`
   files are ASDF components loaded `:serial`; moving them together
   preserves their load order (`package` defines the `geb-docs/docs`
   package that `glossery` and `documentation` enter). The
   `maintainers-guide.org` file is not an ASDF component, so moving it
   has no build effect.
3. Delete the redundant plain-text `README`. The markdown manual at
   `docs/common-lisp/manual.md` supersedes it.
4. Retarget the generator. In
   `docs/common-lisp/documentation.lisp`, rewrite `build-docs` so the
   markdown output is written to `docs/common-lisp/manual.md` via an
   explicit stream (`mgl-pax:document` into a `with-open-file` target)
   and the `:plain` README output is dropped; redirect the companion
   `update-asdf-system-html-docs` `:target-dir` from `docs/` to
   `docs/common-lisp/` for consistency. The rewrite is intended to
   reproduce the existing markdown rendering at the new path, not to
   change document content. Equivalence to the previous
   `update-asdf-system-readmes` output is unverified at authoring time
   because the generator is not rerun in this step; the rewrite may need
   to pass `:pages` and a `:source-uri-fn` (as the HTML call does) to
   reproduce cross-reference and source-link rendering. Confirming the
   output matches is deferred to the next time the generator is run (a
   follow-up, since the Common Lisp build is not exercised here).
5. Create `geb-idris/docs/index.md`. A skeleton index listing the four
   existing Idris documentation files with one-line navigational labels
   derived from their existing titles, plus pointers to
   `../README.md` and `../EXAMPLES.md`. No new explanatory content.
   (`../README.md` and `../EXAMPLES.md` are confirmed present.)
6. Create the root `README.md`. A hand-authored file: a short, dry
   identity statement (what Geb is; active development is Lean, then
   Idris; Common Lisp and Agda are original efforts) followed by a
   categorized index linking to:
   - Lean formalisation (active): `geb-lean/README.md`,
     `geb-lean/docs/index.md`.
   - Idris-2 implementation: `geb-idris/README.md`,
     `geb-idris/docs/index.md`, `geb-idris/EXAMPLES.md`.
   - Agda artefacts (original effort): `geb-agda/README.md`.
   - Common Lisp reference (original, mostly obsolete):
     `docs/common-lisp/manual.md`.

## Commit granularity principle

The total volume of moved bytes is large; the conceptual change is
nil. Every commit does exactly one conceptual thing and is individually
reviewable as either a pure move (optionally with the single reference
update that the move requires), a pure deletion, or a pure
skeleton-creation. No commit both moves content and alters it. Moves
precede the creation of the new root `README.md` so that version
control records the relocation of the old manual as a rename rather
than misattributing it against a freshly created same-named file.

## Generator retargeting detail

`build-docs` currently calls:

```lisp
(mgl-pax:update-asdf-system-readmes
 @index :geb
 :formats '(:markdown :plain))
```

which writes `README.md` and `README` to the ASDF system root. The
replacement writes a single markdown file to `docs/common-lisp/manual.md`
through `mgl-pax:document` and an explicit output stream, and points the
HTML-documentation call (`update-asdf-system-html-docs`) `:target-dir`
at `docs/common-lisp/`. The intent is that the rendered markdown is the
same document at a new path, with the plain-text format dropped.
`update-asdf-system-readmes` is a wrapper whose internal rendering setup
(page configuration, source-link function) is not reproduced by a bare
`mgl-pax:document` call; the rewrite may therefore need to pass `:pages`
and a `:source-uri-fn` to match. Because the generator is not rerun in
this step, the output is not verified here; verifying that the regenerated
`manual.md` matches expectations is a follow-up.

## Markdownlint and doctoc

- Files we author in this step (root `README.md`,
  `geb-idris/docs/index.md`, this spec, the plan) pass
  `markdownlint-cli2` against `geb-lean/.markdownlint-cli2.jsonc`,
  verified locally.
- `docs/common-lisp/manual.md` is generator output, not authored here;
  it is not linted or reformatted.
- Authored documents with more than one `##` heading carry a doctoc
  table of contents, generated locally.

## Process

- Branch: `docs/readme-docs-revamp`, off `main`.
- Spec and plan live under `docs/superpowers/specs/` and
  `docs/superpowers/plans/` at the monorepo root.
- Spec and plan are adversarially reviewed before execution.
- Execution proceeds commit-by-commit under the granularity principle.
- No `jj git push` occurs without line-by-line user review.

## Follow-ups (out of scope)

- Authoring new explanatory content in `geb-lean/docs/index.md`, in new
  Lean documentation files, and in the Idris index and its files.
- Extending CI (`.github/workflows/markdown-lint.yml`) and
  `geb-lean/scripts/pre-push.sh` doctoc and markdownlint coverage to
  parent-level markdown, with a markdownlint configuration scoped to the
  monorepo root and an ignore entry for `docs/common-lisp/`.
- `geb-lean/README.md` describes `../README.md` as "the user-facing
  manual"; after this step the root file is an index and the manual has
  moved to `docs/common-lisp/manual.md`. The link stays valid; only the
  surrounding prose drifts. Reconciling it belongs to the later
  content-authoring phase, where `geb-lean` documentation is edited.
