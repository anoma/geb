- Defect: as stated, `hval : âˆ€ i, i â‰¤ N â†’ f i â‰¤ B` quantifies over

  the whole table; nothing in the `foldBTLOnCode` proof shape

### R1-N5 â€” the Task 6.4.12.8 smoke test exercises no off-gate position

With gate `i â‰¤ code` and the `bprod` running over `j < idxBound + 1`
where the natural `idxBound` is the input itself, every checked
position is gated, so the `condN`-to-`1` off-gate path of the
predicate body is untested by the smoke test. A gate such as
`i â‰¤ code / 2` (extraction still gated) would cover both paths.

### R1-N6 â€” heartbeat guidance is given for Task 6.4.12.1 only

The `foldBTLOnCode` precedent needs `maxHeartbeats 800000`
(`ERTreeArith.lean:734`). Task 6.4.12.8's correctness lemma wraps the
predicate body in an additional `condN` gate over `sane`, so its
elaboration cost is at least that of the 1-D case; the plan flags the
raise only in Task 6.4.12.1.

### R1-N7 â€” the `stepBoundER` route deviation is unrecorded

The gate recipe (wave 11) routes `stepBoundER` through
"`LawvereERTetration` precedents"; the plan uses `ERMor1.towerER` +
`interp_towerER` (`GebLean/LawvereERBoundComputable.lean:230,240`).
The plan's blanket precedence clause covers the disagreement, and the
`towerER` route is the more direct one, but the deviation is the kind
the Execution notes could name explicitly.

### R1-N8 â€” gated-index containment needs pair monotonicity

`Nat.pair d m â‰¤ Nat.pair c c` for gated `d + m â‰¤ c` (required so the
`bprod` over `j < idxBound + 1` covers every gated position, Decision
Q8) needs monotonicity of `Nat.pair` in both arguments. Mathlib
provides `Nat.pair_lt_pair_left`/`Nat.pair_lt_pair_right`
(`Mathlib/Data/Nat/Pairing.lean:112,126`), but the plan's consumed
Szudzik cluster lists only the strictness and additive lemmas.
