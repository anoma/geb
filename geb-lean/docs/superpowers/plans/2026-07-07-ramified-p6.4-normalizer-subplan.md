## Decisions recorded by this mini-sub-plan (for adversarial + user review)

The spec's D1-D6 stand. The plan adds:

### Decision P1 — a reference ℕ layer between `detStep` and the ER terms

Every ER deliverable `X : ERMor1 k` is built against a plain Lean reference
function on ℕ (`stepCode`, `decodeWord`, `codeConc`, `codeBbRep`, the code
detectors), with two lemma families: interp agreement
(`X.interp ![…] = refFn …`, discharged conditionally on the bounded-recursion
trace bounds) and image commutation (`refFn (codeTm t) = codeTm (fn t)`,
pure combinatorics with no ER content). Reason: it separates the arm-by-arm
mirror combinatorics (spec §6.1, the largest cluster) from the
bounded-recursion plumbing, letting each be reviewed and salvaged
independently; precedent `Nat.tuplePack`/`ERMor1.tuplePack` and
`Nat.simRecVec`/`simrec`.

### Decision P2 — `detStep` factors through a rank-targeted worker

`detStep` is a stateless total function that recomputes its dispatch at
every call: `detStep t` reads `q := betaRedexRank t`; if `q > 0` it applies
the β worker `detStepAt q`; else if `hasIota t` it applies the ι worker
`detIotaStep`; else it returns `t`. `detStepAt q` and `detIotaStep` are
`PolyFix.ind` folds implementing the child-priority descent (spec §4.1-4.2).
Reason: statelessness matches the ER step (one code in, one code out, no
threaded state), and the worker factoring gives the cycle lemmas (N1) a
fixed-`q` induction handle; rank recomputation is a computable fold, cost
immaterial (footnote 10).

### Decision P3 — code-level recursion by strong recursion on the code value

`stepCode`, the code detectors, and `decodeWord` recurse by strong recursion
on the code (`Nat.strong_induction_on` or fuel = the code itself): child
codes are strictly smaller than op-node codes (N2). The ER realizations use
the course-of-values pattern of `treeFoldOnCode`/`mutuFoldOnCode`
(`GebLean/LawvereTreeERArith.lean:23,149`): an inner `boundedRec` whose
state packs the table of already-computed values, with the trace bound
discharged from the Task 6.4.8 cluster. Reason: the only ER recursors are
bounded; strong recursion on codes is the shape those precedents already
realize.

### Decision P4 — the adequacy anchor is `prop13_elementary`'s applied term

`buildCode` builds `codeTm (app' (barTm F) (bbRep a (barTy τ)))` and the
adequacy right-hand side is
`freeAlgToNat (appEval (sourceApp F (sourceWord a τ)) finZeroElim)` at
`a := natToFreeAlg (v 0)` — exactly the two sides of `prop13_elementary`
(`Normalization.lean:3103-3110`). Reason: the identification lemmas
(`oneEval_reduces` transport, `normal_closed_o_eq_conc`,
`appEval_sourceWord`) are landed for exactly this shape; any other anchor
would re-prove 6.3 content. This decision records the spec's bbRep
correction (spec §5.3, §6.3, commit `0be2bff3`).

### Decision P5 — mirror statements are generic over `Γ` and `s`

`stepCode_codeTm` (and the detector mirrors) are stated for all
`t : Binding.Tm (oneLambdaSig natAlgSig) Γ s`, not only closed terms.
Reason: `detStep` descends under binders, so the commutation induction
passes through open terms; no detector consulted by the step reads a
variable's sort (redex roots are op nodes; their sorts come from op tags),
so the code needs no context input and the generic statement costs nothing
(spec §6.1 latitude, exercised now).

### Decision P6 — clock statements are split, not packaged

Layer 1 states `detIter_normal` (the deterministic clock); layer 3 states
`normRun_interp` (ER iteration = `detIter` on codes) and the composed
corollary `normRun_normal` (at the Lemma 12 clock value the output codes a
`Normal` term). Reason: the parent interface's `normalizer_clock` is the
composition; splitting lets the term-level and ER-level halves be reviewed
and consumed independently (6.3's `lemma12`/`lemma12_clock` precedent).

## Mathematical notes (binding elaboration)

### N1 — the deterministic cycle and its clock

Write `q := betaRedexRank t`. The β worker `detStepAt q` performs
child-priority descent: at an op node, recurse into the first child `c`
with `betaRedexRank c = q`; if no child qualifies and the node satisfies
`topBetaRank = q`, contract the root redex. Consequences, proved in Task
6.4.3 and consumed by 6.4.4:

1. Contracting a root redex `app' (lam' b) N` under the no-qualifying-child
   guard: the guard on the `lam'` child gives `betaRedexRank b ≤ q - 1`
   (the child's rank equals the body's) and the guard on the argument
   child gives `betaRedexRank N ≤ q - 1`. The contracted redex has arrow
   sort `RType.arrow σ τ` with `RType.ord (RType.arrow σ τ) ≤ q`, and
   `RType.ord σ < RType.ord (RType.arrow σ τ)` gives
   `RType.ord σ ≤ q - 1`; `betaRedexRank_instantiate₁_le` then bounds the
   reduct's rank by
   `max (betaRedexRank b) (max (betaRedexRank N) (RType.ord σ)) ≤ q - 1`
   — the contracted subtree is clean. This is a subtree-local statement,
   not a global count: when the reduct `instantiate₁ N b` is an
   abstraction and the parent node is an application of matching rank,
   the contraction uncovers a new rank-`q` redex at the parent. The
   uncovering is the reason 6.3's `beta_cycle` carries the shape clause
   `IsLam t' → IsLam t ∨ RType.ord s ≤ q` and contracts the root after
   processing the children; `det_cycle` (Task 6.4.4) carries the same
   clause, and per-step redex counting is not used anywhere in this
   plan.
2. The deterministic chain from an op node decomposes compositionally:
   while a child carries a rank-`q` redex, `detStepAt q` equals the
   congruence lift (`appL`/`appR`/`lamBody`) of `detStepAt q` on that
   child (node equations, Task 6.4.1; iterate-level decomposition
   lemmas, Task 6.4.3). Iterating therefore concatenates the children's
   deterministic chains in argument order, then performs the root
   contraction once the children are exhausted (the uncovered-parent
   contractions of item 1 are exactly these root steps, fired at the
   parent's own level of the induction) — the same shape as 6.3's
   `beta_cycle` structural induction, with deterministic sub-chains in
   place of existential ones and at most one root contraction per node.
   The height and size bookkeeping of 6.3 notes N3/N5 transfers with the
   same expressions: per cycle, the chain length is `≤ Tm.size t` (node
   count) and height grows to at most `2 ^ Tm.height t`, within the
   `stepWithin`-style size ceiling.
3. Cycle assembly is 6.3's downward induction (`beta_normalize` shape):
   ranks `q, q-1, …, 1`, then the ι phase. In the ι phase `detIotaStep`
   preserves `betaRedexRank = 0` (the `exists_iota_step_of_hasIota`
   inventory: ι reducts of β-normal terms are β-normal) and strictly
   decreases `Tm.size`, so it terminates within the size ceiling. The total
   is the `lemma12_clock` value
   `(redexRank t + 1) * tower (redexRank t + 1) (Tm.height t)`, fixed by
   the spec for interface stability. Iterating past normality is the
   identity (`detStep_normal`), so `detIter` at any clock value at or above
   the bound lands the same `Normal` term.

### N2 — Szudzik inequalities and structural recursion on codes

With `Nat.pair` (mathlib Szudzik: `if a < b then b * b + a else
a * a + a + b`): `Nat.left_le_pair`/`Nat.right_le_pair` give
`a, b ≤ Nat.pair a b`. Op-node codes are `Nat.pair 1 payload` with
`payload ≥ 1`; `Nat.pair 1 p > p` for all `p ≥ 1` (case split on `1 < p`),
and each child code is `≤ payload` by chained `left_le_pair`/
`right_le_pair`, hence strictly below the node code. Variable leaves are
terminal. Strong recursion on the code value is therefore well-founded for
every structural fold in this plan; Task 6.4.6 lands the inequality cluster
once and all later folds cite it.

### N3 — append-at-end de Bruijn: substitution on codes needs no shifting

`instantiate₁ (e : Tm Γ a) : Tm (Γ ++ [a]) s → Tm Γ s` substitutes the
variable at index `|Γ|`. Under a `lam` binder the context extends at the
end (`(Γ ++ [a]) ++ [σ]`), so the target index is unchanged and the
substituend `e` (whose variables index below `|Γ|`) needs no adjustment —
capture is impossible by index arithmetic alone. The code-level
`subCode j e : ℕ → ℕ` therefore maps variable codes by
`i < j ↦ i`, `i = j ↦ e`, `i > j ↦ i - 1`, recurses through op nodes with
`j` and `e` both constant, and mirrors `instantiate₁` at `j = |Γ|`
(Task 6.4.9). No renaming function on codes is required.

### N4 — course-of-values folds on codes

The ER realization of a strong recursion on codes follows
`treeFoldOnCode`/`mutuFoldOnCode` (`LawvereTreeERArith.lean`): an
`ERMor1.boundedRec` whose recursion variable runs up to the input code and
whose state packs the table of values at all smaller codes (Szudzik
sequence packing, `seqPack`/`seqAt` shapes), reading child values out of
the table. The `h`/`h_mono` obligations of `boundedRec_eq_natRec_of_bounded`
are discharged by a monotone bound on the packed table, assembled from the
per-value bounds of Task 6.4.8. Task 6.4.11's first step is a read of the
precedent and a recorded adaptation note (the 6.2/6.3 re-checkpoint
pattern) before any ER code is written.

### N5 — the `normRun` trace ceiling

`normRun`'s outer `boundedRec` iterates `normStep` from `codeTm t`. Every
term on the deterministic chain from `t` has size within the 6.3
`stepWithin` ceiling and height within `tower q h`-shaped bounds (N1), and
its sorts are drawn from the source term (spec §5.4 sort stability), so by
the Task 6.4.8 envelope every code on the chain is `≤ codeCeil t` for a
single monotone tower-shaped expression in `Tm.size t`, `Tm.height t`, and
the source sort payload. `codeCeil` is constant in the recursion variable,
so `h_mono` is trivial and `h` is the constant bound.

### N6 — `buildCode` and `clockER` structure

`natToFreeAlg v` is the `v`-fold `true`-constructor word. Its `bbRep` at
`barTy τ` is a fixed `lamSpine` prefix (a function of `τ` only) over a
`v`-fold constructor-variable application spine, so
`codeBbRep τ v` is a `v`-fold iteration of a fixed code transformer around
a fixed core — one `boundedRec` with the Task 6.4.8 envelope as bound.
`clockER` computes `(q_F + 1) * tower (q_F + 1) (h_F + v + c)` with
`q_F`/`h_F`/`c` meta-level constants of `F` (the `prop13_elementary` proof
already constructs such a pair at `Normalization.lean:3111-3115`): the
tower is a `(q_F + 1)`-fold composition of the ER exponential `2 ^ x`
(a `bprod` over a constant, `LawvereERTetration.lean` precedents), times
the constant, over the successor-and-addition atoms.
