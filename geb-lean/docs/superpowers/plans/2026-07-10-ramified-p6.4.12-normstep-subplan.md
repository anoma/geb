**Goal:** Realize the reference deterministic step on codes as an ER
morphism — `normStep : ERMor1 1` with the unconditional interpretation
lemma `normStep_interp (n : ℕ) : normStep.interp ![n] = stepCode n` — by
decomposing the parent plan's Task 6.4.12 into twelve dependency-ordered
waves, per the re-checkpoint recorded at the 6.4.12 gate (the execution
handoff `docs/superpowers/notes/2026-07-08-ramified-p6.4-execution-handoff.md`
§"The Task 6.4.12 escalation").

**Architecture:** One fresh generic course-of-values table-search
combinator over Szudzik-coded trees (`ERMor1.cvRec`, plus a gated
two-dimensional variant `ERMor1.cvRecGated`), homed in
`GebLean/Utilities/` beside `ERMor1.foldBTLOnCode` as the missing general
concept; nine table folds realizing the `stepCode` ingredients (seven
one-dimensional, two gated two-dimensional), each carrying its own
universal majorant proved at the ℕ level; a
`boundedRec`-iterated substituend weakening; and a `condN`/`minN`
dispatch-and-clamp assembly reproducing `stepCode`'s definition literally.
Every fold's interpretation lemma is unconditional (the majorants hold on
arbitrary codes, not only term codes), so the final `normStep_interp`
holds for all `n` by composition; no `codeTm` mirror appears at this
layer (that is Task 6.4.13's job).

## Decisions recorded by this mini-sub-plan (for adversarial + user review)

### Decision Q1 — a fresh generic combinator, not a `foldBTLOnCode` instance

`ERMor1.foldBTLOnCode` and its five supporting lemmas are hardwired to one
recursion shape: even code `2m` is a leaf, odd code `2p + 1` combines the
two children `(Nat.unpair p).1`/`.2`, and the node (`foldBTLExpected`) is
a fixed `condN` over `j % 2` reading exactly those two β-positions. No
`stepCode` ingredient has that shape: `ordCode` dispatches on
`shapeCode ∈ {1, 2}` and recurses into sort-code children; the term-code
folds dispatch on the top kind bit and the operation kind bit and read
children at the fixed unpair depths of `child0Code`/`child1Code`. The
fixed node and the fixed child-position reads inside
`foldBTLPred_trace_match`'s induction cannot be reused. A generic
combinator whose node abstracts both is required; it belongs in
`GebLean/Utilities/` beside `foldBTLOnCode` as a reusable abstraction
(the parent plan's N4 latitude — adapt the pattern, not the instance —
exercised at the recorded escalation).

### Decision Q2 — the node reads the candidate

`cvRec`'s node morphism has arity `k + 3` at slots
`(i, cand, code, params…)`: it computes the recursion equation's
right-hand side at table index `i`, performing its own β-reads
`β(cand, p) = cand.unpair.1 % (1 + (p + 1) * cand.unpair.2)` at child
positions it computes from `i` (via `ERMor1.beta` or the literal
mod-read composition). This frees the combinator from any fixed node
shape; the per-client correctness input becomes a node-faithfulness
hypothesis (M2). Same-function recursive references go through the
β-table; previously realized helpers are composed as ordinary full calls
at computed inputs.

### Decision Q3 — Bool-valued detectors realize as `Bool.toNat`

The ℕ-level detectors returning `Bool` (`isLamCode`, `conHeadedCode`,
`iotaSpineCode`, `topIotaCode`, `hasIotaCode`) realize as ER morphisms
whose interpretation equals `(…Code c).toNat`, with `1`/`0` the
dispatch-visible values (`condN`/`signN` consume them directly). This
matches the `boolEqNat` convention of `LawvereERBool`.

### Decision Q4 — unconditional interpretation lemmas via universal majorants

Every fold's `valBound` is a universal majorant of the ℕ-level function
on arbitrary codes (M4), not merely on term codes, so every interp lemma
is unconditional (`∀ n`, no term-code hypothesis). The `min`-clamp of
`stepCode` is therefore NOT needed to bound the fold tables (the per-fold
`valBound`s do that, inside each search); it is reproduced solely in the
final assembly because `stepCode`'s ℕ-level definition carries it, making
`normStep_interp` hold for ALL `n` by composing unconditional pieces.
The conditional fallback contract (interp lemmas hypothesized on
`n`-bounded inputs) is explicitly not adopted — the obstacle at the gate
was volume, not provability.

### Decision Q6 — determinacy is client-supplied in the gated variant

For 1-D folds the node-faithfulness hypothesis in the form
"`∀ i ≤ N`, any candidate whose β-reads agree with the reference table at
all positions `p < i` has `node.interp … = f i`" yields both existence
(the β-witness of `Nat.bounded_beta_exists` agrees everywhere) and
determinacy (strong induction on the position), because every child
position a node reads is strictly below its index (Szudzik: children of
a code sit strictly below it). For the 2-D folds the descent order is
not the numeric index order — at an abstraction descent
`Nat.pair (d + 1) body` may exceed `Nat.pair d m` — so `cvRecGated`
takes determinacy as an explicit client hypothesis (M3), discharged per
instance by strong induction on the code component `m` (which the
pairing strictly decreases), with the gate keeping every reachable
position within the checked range (reachability preserves `d + m ≤ c`).

### Decision Q7 — the iterated substituend is a `boundedRec`, not a table

`subCode`'s abstraction descent replaces the substituend `e` by
`shiftCode j e`; the 2-D table for `subER` therefore stores, at index
`Nat.pair d m`, the value `subCode j ((shiftCode j)^[d] e) m`. The
iterate `(shiftCode j)^[d] e` is realized as `shiftIterER` by
`ERMor1.boundedRec` over `shiftER` (one step per depth increment),
discharged by `boundedRec_eq_natRec_of_bounded` — the one non-table
recursion of the decomposition, matching that combinator's exact shape.

### Decision Q8 — 2-D keying, gate, and extraction

The two parameter-varying recursions (`subCode`, `betaStepCode`) become
gated 2-D folds with index `i = Nat.pair d m` (`d` the abstraction
depth, `m` the code), gate `sane i = (d + m ≤ c).toNat` for input code
`c`, index bound `idxBound = Nat.pair c c`, and extraction at
`extractAt = Nat.pair 0 c` (depth `0` = the input level). Indices
outside the gate would otherwise impose equations whose reads escape
the checked range; the reference table is defined as `0` off the gate,
which makes the combinator's unconditional value-bound hypothesis (M3)
trivial there — the substantive bounds are the gated entries'.

## Mathematical notes (binding elaboration)

### M1 — the β-read convention and the search envelope

Throughout, the β-read of candidate `cand` at position `p` is
`β(cand, p) = cand.unpair.1 % (1 + (p + 1) * cand.unpair.2)` (the
`ERMor1.beta`/`betaOnCandFold` convention). A candidate encoding a table
`s : Fin (N + 1) → ℕ` with values `≤ B` exists below
`((N + B + 3).factorial) ^ ((N + B + 3).factorial) + 1`
(`Nat.bounded_beta_exists` + `Nat.pair_lt_boundedRecRange`); the search
range morphism must therefore evaluate `boundedRecRange` with the index
bound in its counter slot (`N = code` for `cvRec`; `N = idxBound` for
`cvRecGated`) and the client `valBound` as its bound argument.

### M2 — `cvRec` correctness (binding statement)

```lean
theorem ERMor1.interp_cvRec_of_bounded {k : ℕ}
    (node : ERMor1 (k + 3)) (valBound : ERMor1 (k + 1))
    (code : ℕ) (ctx : Fin k → ℕ) (f : ℕ → ℕ)
    (hval : ∀ j, j ≤ code → f j ≤ valBound.interp (Fin.cons j ctx))
    (h_mono : ∀ j, j ≤ code →
      valBound.interp (Fin.cons j ctx) ≤
        valBound.interp (Fin.cons code ctx))
    (h_node : ∀ i, i ≤ code → ∀ cand,
      (∀ p, p < i →
        cand.unpair.1 % (1 + (p + 1) * cand.unpair.2) = f p) →
      node.interp
        (Fin.cons i (Fin.cons cand (Fin.cons code ctx))) = f i) :
    (ERMor1.cvRec node valBound).interp (Fin.cons code ctx) = f code
```

`h_node` discharges existence through `Nat.bounded_beta_exists` exactly
as in `foldBTLOnCode` (the witness agrees with `f` everywhere, hence at
all `p < i`), and determinacy by strong induction on the position (every
read is below the index, Decision Q6). Companion (3 lines, the
`interp_foldBTLOnCode_le_bound` copy):

```lean
theorem ERMor1.interp_cvRec_le_bound {k : ℕ}
    (node : ERMor1 (k + 3)) (valBound : ERMor1 (k + 1))
    (ctx : Fin (k + 1) → ℕ) :
    (ERMor1.cvRec node valBound).interp ctx ≤ valBound.interp ctx
```

### M3 — `cvRecGated` correctness (binding statement)

`sane : ERMor1 (k + 2)` at slots `(i, code, params…)` with values in
`{0, 1}`; `idxBound extractAt valBound : ERMor1 (k + 1)` at slots
`(code, params…)`. The per-position predicate body is
"`sane = 1` implies the β-read matches the node" (a `condN` sending
ungated positions to `1` inside the `bprod` over `j < idxBound + 1`).

```lean
theorem ERMor1.interp_cvRecGated_eq {k : ℕ}
    (node : ERMor1 (k + 3)) (sane : ERMor1 (k + 2))
    (idxBound extractAt valBound : ERMor1 (k + 1))
    (code : ℕ) (ctx : Fin k → ℕ) (f : ℕ → ℕ)
    -- notation below: N, B, X abbreviate the interps of idxBound,
    -- valBound, extractAt at (Fin.cons code ctx); S i abbreviates
    -- sane.interp (Fin.cons i (Fin.cons code ctx))
    (h_sane : ∀ i, i ≤ N → S i ≤ 1)
    (hval : ∀ i, i ≤ N → f i ≤ B)
    (h_node : ∀ i, i ≤ N → S i = 1 → ∀ cand,
      (∀ p, p ≤ N →
        cand.unpair.1 % (1 + (p + 1) * cand.unpair.2) = f p) →
      node.interp
        (Fin.cons i (Fin.cons cand (Fin.cons code ctx))) = f i)
    (h_det : ∀ cand,
      (∀ i, i ≤ N → S i = 1 →
        cand.unpair.1 % (1 + (i + 1) * cand.unpair.2) =
          node.interp
            (Fin.cons i (Fin.cons cand (Fin.cons code ctx)))) →
      cand.unpair.1 % (1 + (X + 1) * cand.unpair.2) = f X)
    (h_ext : X ≤ N) (h_fext : f X ≤ B) :
    (ERMor1.cvRecGated node sane idxBound extractAt valBound).interp
      (Fin.cons code ctx) = f X
```

The exact hypothesis packaging (`let`-abbreviations versus explicit
interps, whether `h_fext` is folded into `hval` + `h_ext`) carries
implementer latitude; the hypothesis CONTENT above and the conclusion
are binding. `hval` quantifies over every index, gated or not: the 2-D
clients define their reference tables as `0` off the gate (Decision
Q8), which makes the off-gate instances trivial, and
`Nat.bounded_beta_exists` needs the whole table `[0, N]` bounded.
`h_det` is the per-instance analogue of `foldBTLPred_trace_match`
(Decision Q6), proved by each 2-D client by strong induction on the
code component of the index. Gated-index containment (every gated
`Nat.pair d m` with `d + m ≤ c` sits at or below
`idxBound = Nat.pair c c`) uses the monotonicity of `Nat.pair` in both
arguments (mathlib `Nat.pair_lt_pair_left`/`Nat.pair_lt_pair_right`,
`Mathlib/Data/Nat/Pairing.lean`).

### M4 — the majorant ledger (new ℕ-level lemmas, all binding)

Proof route for the tower bounds: strong induction on the code through
the node-equation families, with `+9`-per-node absorption — each node
rebuild costs at most four `pair_le_tower_two` pairings (`+8` in the
tower argument), paid for by the child codes being at least one smaller;
the abstraction cases of `subCode`/`betaStepCode` also pay the depth
increment, so their code coefficients exceed the depth coefficients by
at least `9`.

| lemma (home) | statement |
| --- | --- |
| `ordCode_le_self` (`CodeTm.lean`) | `ordCode n ≤ n` |
| `topBetaRankCode_le_succ` (`CodeNormalizer.lean`) | `topBetaRankCode c ≤ c + 1` |
| `betaRankCode_le_succ` (`CodeNormalizer.lean`) | `betaRankCode c ≤ c + 1` |
| `iotaStepCode_le_tower` (`CodeNormalizer.lean`) | `iotaStepCode m ≤ tower 2 (9 * m + 9)` |
| `shiftCode_le_tower` (`CodeNormalizer.lean`) | `shiftCode j m ≤ tower 2 (9 * m + 9)` |
| `shiftCode_iterate_le_tower` (`CodeNormalizer.lean`) | `(shiftCode j)^[d] m ≤ tower 2 (9 * m + 9 * d + 9)` |
| `subCode_shift_iterate_le_tower` (`CodeNormalizer.lean`) | `subCode j ((shiftCode j)^[d] e) m ≤ tower 2 (18 * m + 9 * e + 9 * d + 18)` |
| `betaStepCode_le_tower` (`CodeNormalizer.lean`) | `betaStepCode q d m ≤ tower 2 (27 * m + 9 * d + 18)` |

The exact constants are binding as ceilings the fold `valBound`s must
dominate on the gate; an implementer discovering a slack constant may
tighten a lemma only together with its consuming `valBound` (both in the
same task), never loosen one.

### M5 — the fold table (the interior seam, binding layout)

Each helper is its own fold whose node composes the previously realized
helpers as ordinary full calls at computed inputs; only same-function
recursive references go through the β-table.

| fold | arity (slots) | index | value at index | `valBound` |
| --- | --- | --- | --- | --- |
| `ordER` | 1 (`c`) | 1-D `i` | `ordCode i` | `c` |
| `conHeadedER` | 1 | 1-D | `(conHeadedCode i).toNat` | `1` |
| `iotaSpineER` | 1 | 1-D | `(iotaSpineCode i).toNat` | `1` |
