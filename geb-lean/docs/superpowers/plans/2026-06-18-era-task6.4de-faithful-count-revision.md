## Why this revision exists: where we departed from the paper

The construction is a pure transcription of paper-proven mathematics
(arXiv:2407.12928 § 4 / Corollary 3.6 for the count; arXiv:2606.09336
Claims 1-5 for its application). It must invent no parallel machinery.

The count primitive of arXiv:2407.12928 (Lemma 3.3 / Cor 3.6) is: for a
predicate `P(n⃗, x⃗)` over parameters `n⃗` and cube variables `x⃗`, an
arithmetic term computing `#{a⃗ ∈ [0, t)^k : P(n⃗, a⃗) = 0}`. The cube
variables `x⃗` are whatever one is counting over; they are arbitrary.

This primitive is already realised faithfully in the repository as the
general combinator

```lean
def eraCount {p k : ℕ} (s : SosSystem (p + k)) (tTerm wTerm : ETm p) : ETm p
theorem eraCount_eval … -- count via the Lemma-3.5-reduced predicate
theorem reducedCount_eq … -- collapse the reduced predicate to SosSystem.eval s
theorem predCount_side_eq … -- collapse the enlarged side back to the base side
```

whose composition gives, for any `s : SosSystem (p + k)` carrying the three
structural invariants (`PolyExpZero`, `CoeffVarProduct`, `BasePaired`) and a
coordinate bound,

```text
eval (eraCount s tθ (w+1)) ctx
  = #{x⃗ ∈ cubePoints k tBase : SosSystem.eval s (Fin.append ctx x⃗) = 0}.
```

That is precisely the Cor 3.6 count over arbitrary cube variables `x⃗`. Its
hypotheses are general; nothing ties them to a `diophOf` graph. (Confirmed
from the signatures of `eraCount_eval`, `reducedCount_eq`, `predCount_side_eq`
in `GebLean/EraHistCodeTerm.lean`.)

The departure is one layer up, at the Task 6.4c capstone `eraCountPred`:

```lean
def eraCountPred {n : ℕ} (τ : ETm n) : ETm n :=
  eraCount (reindexSys τ) (eraTheta …) (eraW … +ᵉ eraNumeral 1)
```

`reindexSys τ` is the `diophOf` graph of a *function* `τ`, re-indexed so the
cube variables are the **output slot together with the witnesses**. By the
soundness and unique-witness conjuncts of `DiophEnc.Encodes`, that fibre is a
singleton: `eraCountPred τ` evaluates to `1` for every total `τ`. The papers
never count a function graph's output fibre. Their Claims 3 and 5 feed the
Section-4 count a **predicate** whose cube variables are the genuine unknowns:
`(j, z⃗, λ⃗)` for Claim 3, `(ω₁, ω₂, μ⃗)` for Claim 5.

So `eraCountPred` is a special case the papers do not use; the faithful
consumer of Cor 3.6 is the general `eraCount` chain, applied to the system of
the master relation. Tasks 6.4d/6.4e therefore consume the general chain and
bypass `eraCountPred`. The deferred "solCount corollary" of the packM handoff
(notes/2026-06-18, lines 117-128) is subsumed: it was an attempt to recover a
genuine count from `eraCountPred`'s output fibre, which is unnecessary once the
genuine unknowns are placed on the cube side directly.

## The one transcription gap: inputs as unknowns of `E`

The existing `reindexSys` moves only the `diophOf` **output** to the cube
side; the inputs stay parameters. The papers' relation `E(n, x, ȳ)(j, z⃗, λ⃗)`
treats chosen **inputs** (`j`; later `ω₁, ω₂`) as unknowns of `E` as well.
Completing that transcription is the only new structural object:

- **`reindexInputSys`** — the faithful sibling of `reindexSys`. Given a
  `diophOf`-encoded predicate term `pred : ETm (p + r)` (parameters `p`,
  counted inputs `r`), re-index its system so the `r` counted inputs, the
  output slot, and the witnesses all become cube coordinates, with the output
  slot pinned to `0` (an appended `sqDist [output] []` atom, since the empty
  `SimpleSum` evaluates to `0`). The result is a `SosSystem (p + (r + 1 +
  witArity))` whose zeros over the cube are exactly
  `{(a⃗, 0, w⃗) : pred(ctx, a⃗) = 0 ∧ w⃗ the unique witness}`.

  Structurally this is a *re-grouping plus a pin atom*, not a slot-permuting
  embedding. `DiophEnc.ctx = Fin.append (Fin.snoc ρ y) w` already orders the
  variables as `[inputs (p+r), output, witnesses]`. We want the parameter side
  to be the `p`-prefix of the inputs and the cube side to be the suffix
  (`moved-inputs ⧺ output ⧺ witnesses`), so the param/cube split is only moved
  to a different position — there is no reordering. The re-grouping is an
  associativity iso `Fin ((p+r)+1+witArity) ≃ Fin (p + (r+1+witArity))`. At
  execution, evaluate the cheapest realisation (in order of preference):
  `finCongr` of `Nat.add_assoc`; the `reindexEmb` `Fin.addCases` pattern; or
  the existing `spliceEmb`/`SosSystem.spliceWeaken`
  (`EraDiophantine.lean:635-649`, with `spliceEmb_injective` already proven) —
  reuse the latter if it fits, per the reuse rule. The output-pin atom is the
  one genuinely new fragment beyond `reindexSys`. The three invariants transfer
  through `SosSystem.weaken_*` and `SosSystem.*_append` (the pin atom is the
  mirror image of the `diophZero` atom `sqDist [] [var]`, whose invariants are
  already proven); the coordinate bound is the analogue of
  `reindexSys_coord_bound`, using `diophOf_bound` for the witnesses, the output
  pinned to `0` (so `< 1 ≤ tBase`, which needs `0 < tBase` — see the base-bound
  positivity obligation in 6.4d-3), and the per-input box bound for the counted
  inputs.

This is not invention: it is the input-side counterpart of the output-side
`reindexSys` already in the file, and it realises the papers' explicit choice
of which letters of `E` are unknowns. Everything else is reuse.

## Task 6.4d (revised): `eraHitCount`

Realise `hitCount step A x n = #{j < n : hitsAt step A x j}` as an `Era` term.

**Parametrization (correction; binding).** The downstream contract forces `n`
(the loop bound) to be an input *variable*, not an ambient numeral:
`ERMor1.interp_bsum` (`LawvereER.lean:130`) is
`natBSum (ctx 0) (fun i => f.interp (Fin.cons i (Fin.tail ctx)))` — slot `0` is
the loop bound and slots `1..k` are ambient parameters — and `eraRec` (Task 6.5)
reads off `recSeq init step (ctx 0) = histCode … (ctx 0) / A^(ctx 0)`
(`recurrence_readoff`, `EraHypercube.lean:401`) at that variable bound. So
`eraHitCount`/`eraHistCode` are `(k+1)`-ary **formers**, identical in
parametrization to `eraRec`:

- `n` is the loop-bound slot (input), the ambient `k` parameters thread through;
- the recurrence `step` and the coding base `A` are **supplied `Era` terms**
  (`stepTm`, `ATerm`), not numerals — `stepTm` computes `step j v`
  (`(j, value, params)`), `ATerm` computes `A(n, params)`; `init` is a numeral
  (`0` for `bsum`, `1` for `bprod`);
- the `eval` lemmas carry the `recurrence_readoff` bound hypothesis
  `∀ j ≤ n, recSeq init step j < A`.

Building them `(k+1)`-ary from the start (the `k` params are passive context
that `reindexInputSys`/`eraCount` thread for free) avoids a rework at Task 6.5,
since `eraHistCode` *is* `eraRec`'s core. The exact slot order
(`Fin.cons`/`Fin.tail` bookkeeping) is pinned at execution against
`interp_bsum`. The steps below state the bare (`k = 0`, single loop-bound input
`n`, with `stepTm`/`ATerm` over that context) shape for readability; the
implementer threads the `k` ambient params uniformly.

The per-`j` hit gap is the predicate the Section-4 count is applied to.

- [x] **6.4d-1: the hit-gap predicate term `hitGapTm`** over the params
  `(x, n)` (bare; plus the `k` ambient params) and the counted input `j`, with
  `eval hitGapTm (x, n, j) = 0 ↔ hitsAt step (A n) x j ∧ j < n`, where the
  coding base `A = eval ATerm (n, params)` appears as a subterm (substitute
  `ATerm` for the base). Build from existing Era constructors:
  `digit(x, j) := (x /ᵉ (ATerm ^ᵉ j)) %ᵉ ATerm`;
  `lhs := stepTm` applied to `(j, digit(x, j), params)`;
  `rhs := (x /ᵉ (ATerm ^ᵉ (j+1))) %ᵉ ATerm`;
  `gap := (lhs ∸ᵉ rhs) +ᵉ (rhs ∸ᵉ lhs)` (zero iff `lhs = rhs`, i.e. `hitsAt`);
  `rangeGap := (j +ᵉ 1) ∸ᵉ n` (zero iff `j < n`);
  `hitGapTm := gap +ᵉ rangeGap`. Prove `hitGapTm_eval` and the `↔` (the
  `hitsAt` half uses `Nat.sub_eq_zero` antisymmetry; the range half uses
  `j + 1 ≤ n ↔ j < n`). This is pure Era-term arithmetic; no Diophantine
  content yet.

- [x] **6.4d-2a: `reindexInputSys` re-grouping + pin, with its `eval` bridge.**
  Build the system `reindexInputSys (pred) := (diophOf pred).sys.weaken e ++
  [pin]` for `r = 1` (state it general in `r` if free; `r = 2` is reused in
  6.4e), where `e` is the associativity re-grouping (per the transcription-gap
  section) and `pin := sqDist [SimpleMonomial.var outputSlot] []`. Prove the
  `eval` bridge (the analogue of `reindexSys_eval`: vanishing iff the
  `diophOf pred` graph holds at the re-grouped context with output read from the
  cube, and that output is `0`). The embedding's injectivity is the proof risk;
  prefer `finCongr`/`spliceEmb` reuse to a fresh `addCases` map.

- [x] **6.4d-2b: the three invariant transfers.** `reindexInputSys pred`
  carries `PolyExpZero`/`CoeffVarProduct`/`BasePaired`, via `weaken_*` on the
  `diophOf` invariants and `*_append` with the pin atom (the pin is the mirror
  of `diophZero`'s atom; reuse its invariant lemmas). No bespoke invariant proof.

- [x] **6.4d-2c: the coordinate bound** (analogue of `reindexSys_coord_bound`):
  every cube zero has all coordinates `< tBase`, using `diophOf_bound` for the
  witnesses, the pinned output `= 0 < 1 ≤ tBase`, and the per-input box bound.

- [x] **6.4d-3: the inner system `E_inner` and its count.**
  `E_inner := reindexInputSys (diophOf hitGapTm)` (a `SosSystem (1 + (1 + 1 +
  witArity))`, parameters `= (x)`). `E_inner` is a faithful *re-encoding* of the
  `k = 1` master relation's solution predicate — the digit extraction is done
  arithmetically (`x / A^j % A`) and `diophOf`-reduced, rather than as the
  paper's literal `ECode + Eπ + EF` conjunction; the two share the same set of
  hitting `j`, which is all the count depends on (established by 6.4d-4). Define
  the base side `tBase` as an `Era` term dominating `n` (so `j < n` and the
  pinned output `0` both lie in `[0, tBase)`, giving `0 < tBase`), the output
  `= 0`, and every witness bound — an `eraListSum` of `n`, `1`, and the
  `diophOf hitGapTm` bounds, routed through `eraMajorant` exactly as
  `eraBaseBound` is, so `0 < eval tBase` is inherited (the standing `ht`
  obligation of `eraCount_eval`). Then
  `eraHitCount := eraCount E_inner (eraTheta E_inner tBase) (eraW E_inner
  (eraTheta E_inner tBase) +ᵉ eraNumeral 1)`, and prove the count identity
  `eval eraHitCount ![x] = #{a⃗ ∈ cube : SosSystem.eval E_inner (append ![x]
  a⃗) = 0}` by the same `eraCount_eval → reducedCount_eq → predCount_side_eq`
  chain `eraCountPred_eval` uses (bounds discharged by `eraW_spec`,
  `eraTheta_spec`, and 6.4d-2c's coordinate bound).

- [x] **6.4d-4: the ℕ witness-collapse to `hitCount`.** Prove
  `#{a⃗ ∈ cube : SosSystem.eval E_inner (append ![x] a⃗) = 0} = hitCount step A
  x n`. Via 6.4d-2's `eval` bridge and `diophOf_graph_iff`/`diophOf_unique`,
  each cube zero is `(j, 0, w⃗)` with `w⃗` the unique witness of
  `hitGapTm(x, j) = 0`, so the map `a⃗ ↦ j` is a bijection onto
  `{j < n : hitsAt step A x j}` (`Finset.card_nbij'`, inverse
  `j ↦ (j, 0, the-unique-w⃗)`). Its cardinality is `hitCount` by definition.
  Consumes `hitGapTm`'s `↔` (6.4d-1), `diophOf_unique`, `diophOf_bound`,
  `mem_cubePoints`.

## Task 6.4e (revised): `eraHistCode`

Realise `histCode init step (A n) n` as an `Era` term in the loop bound `n`
(and the ambient params), as the outer Section-4 count of `E₂`
(arXiv:2606.09336 Claims 4-5). With the `(k+1)`-ary former `eraHitCount` in

- [x] **6.4e-1: the outer predicate term `histGapTm`** over the params `(n)`
  (bare; plus the `k` ambient params) and the counted inputs `(ω₁, ω₂)`, with
  `eval histGapTm (n, ω₁, ω₂) = 0 ↔ hitCount step (A n) (ω₁+ω₂+1) n = n
  ∧ (ω₁+ω₂+1) / (A n)^0 % (A n) = init ∧ ω₁+ω₂+1 < (A n)^(n+1)`. Build by
  substituting `x = ω₁ +ᵉ ω₂ +ᵉ 1` into `eraHitCount` (via `Tm.subst`) and
  combining the three conditions as Era `∸`/`+` gaps, mirroring 6.4d-1. Use the
  symmetric gap `(hitCount ∸ n) +ᵉ (n ∸ hitCount)` for the `= n` condition (so
  `hitCount ≤ n`, which holds but is not assumed, is unnecessary).
  **Scale watch (re-checkpoint here if it bites):** `histGapTm` contains the
  full `eraHitCount` term (itself an `eraCount`), so `diophOf histGapTm` recurses
  through the entire count term and its `witArity` may be very large. Correctness
  is unaffected (`diophOf_encodes` is structural over every `ETm`), but the term
  and its proofs may be heavy. If the compile is intractable, pause and report:
  options include factoring the substitution behind an opaque `eraHitCount`
  graph rather than its expanded body.

- [x] **6.4e-2: the outer system and its count** using `reindexInputSys`
  with `r = 2` (the counted inputs `ω₁, ω₂`):
  `E₂ := reindexInputSys (diophOf histGapTm)`; `eraHistCode := eraCount E₂ …`
  exactly as 6.4d-3, with the base side dominating `ω₁, ω₂ < A^(n+1)`.

- [x] **6.4e-3: the ℕ collapse to `solCount`.** Prove
  `#{a⃗ ∈ cube : SosSystem.eval E₂ (append (n,params) a⃗) = 0} = solCount init
  step (A n) n`. The cube zeros are `(ω₁, ω₂, 0, μ⃗)` with `μ⃗` unique; the map

  `diophOf_bound`). This is the genuine count `solCount` (not `1`): the
  unknowns `ω₁, ω₂` are free, and `card_pairs_succ_sum` is reached through
  `solCount_eq_histCode`, not re-derived here.

- [x] **6.4e-4: assemble `eraHistCode_eval`.** State it with the trajectory
