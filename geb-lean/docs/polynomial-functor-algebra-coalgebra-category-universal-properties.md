# Polynomial Functor Algebra and Coalgebra Universal Properties

Let **Set** denote the category of sets and functions.
Let P: **Set** -> **Set** be an accessible polynomial
endofunctor. We examine the category of P-algebras, denoted
P-**Alg**, and the category of P-coalgebras, denoted
P-**Coalg**.

The running example is the functor P(X) = A + X^2,
representing binary trees with leaves labeled by elements
of a fixed set A.

## 1. Initial and Terminal Objects

### 1.1 Algebras

#### 1.1.1 General Case

The forgetful functor U: P-**Alg** -> **Set** strictly
creates limits. Consequently, the terminal algebra is
constructed upon the terminal object in **Set** (the
singleton set 1 = {*}), equipped with the unique structure
map P(1) -> 1.

Because P is accessible, P-**Alg** is cocomplete and
therefore possesses an initial object. The initial algebra
(W, alpha: P(W) = W) represents the type of strictly
well-founded trees (W-types) generated by the signature
of P. By Lambek's Lemma, alpha is an isomorphism.

#### 1.1.2 Example: P(X) = A + X^2

* **Terminal Algebra:** The underlying set is {*}. The
  operations are defined trivially: leaf(a) = \* for all
  a in A, and node(\*, \*) = \*.
* **Initial Algebra:** W is the set of all strictly finite
  binary trees with leaves labeled by A. The isomorphism
  W = A + W^2 dictates that every tree in W is uniquely
  either a distinct leaf a in A or a node joining two
  finite trees from W.

### 1.2 Coalgebras

#### 1.2.1 General Case

The forgetful functor U: P-**Coalg** -> **Set** strictly
creates colimits. Therefore, the initial coalgebra is
constructed upon the initial object in **Set** (the empty
set {}), equipped with the unique structure map {} -> P({}).

Because P is accessible, P-**Coalg** has a terminal object.
The terminal coalgebra (Z, zeta: Z = P(Z)) represents
the type of all observable unfolding behaviors (M-types).
By Lambek's dual lemma, zeta is an isomorphism.

#### 1.2.2 Example: P(X) = A + X^2

* **Initial Coalgebra:** The underlying set is {}. It
  represents a degenerate state machine with an empty
  state space.
* **Terminal Coalgebra:** Z is the set of all finite and
  infinite binary trees with leaves labeled by A. It
  constitutes the maximal state space of all valid binary
  unfoldings over A.

## 2. Free Monads and Cofree Comonads

### 2.1 Algebras (The Free Monad)

#### 2.1.1 General Case

Because P is accessible, the forgetful functor
U: P-**Alg** -> **Set** has a left adjoint
F: **Set** -> P-**Alg**. For any set Y, F(Y) is the free
P-algebra generated by Y. The composition U . F forms
the free monad M on **Set**. F(Y) is constructed as the
initial algebra for the modified functor P_Y(X) = Y + P(X).

#### 2.1.2 Example: P(X) = A + X^2

The free algebra on a set Y is the set of all finite
binary trees where the leaves are labeled by elements
of the disjoint union A + Y. Elements of Y act as freely
inserted, uninterpreted variables within the syntactic
tree structure.

### 2.2 Coalgebras (The Cofree Comonad)

#### 2.2.1 General Case

Because P is accessible, the forgetful functor
U: P-**Coalg** -> **Set** has a right adjoint
G: **Set** -> P-**Coalg**. For any set Y, G(Y) is the
cofree P-coalgebra over Y. The composition U . G forms
the cofree comonad C on **Set**. G(Y) is constructed as
the terminal coalgebra for the modified functor
P_Y(X) = Y x P(X).

#### 2.2.2 Example: P(X) = A + X^2

The cofree coalgebra on a set Y consists of the finite
and infinite binary trees from Z, but with a structural
addition: every valid state observation (every node and
every leaf) is explicitly annotated with an element of Y.
It represents the maximal unfolding behavior of P,
coupled with a continuous stream of outputs in Y at
every observational step.

## 3. Binary Products

### 3.1 Algebras

#### 3.1.1 General Case

Because U: P-**Alg** -> **Set** creates limits, the
binary product of algebras (S1, alpha1) and (S2, alpha2)
is the Cartesian product S1 x S2. The structure map
operates component-wise.

#### 3.1.2 Example: P(X) = A + X^2

The product algebra has the underlying set S1 x S2.
Evaluation occurs in parallel:

* leaf(a) = (leaf1(a), leaf2(a))
* node((x1, x2), (y1, y2)) = (node1(x1, y1), node2(x2, y2))

### 3.2 Coalgebras

#### 3.2.1 General Case

U: P-**Coalg** -> **Set** does not create products.
The binary product of coalgebras (C1, gamma1) and
(C2, gamma2) is the maximal synchronized bisimulation
between the two state spaces. It is constructed
equivalently as the pullback of C1 and C2 over the
terminal coalgebra Z in **Set**.

#### 3.2.2 Example: P(X) = A + X^2

The product coalgebra is the maximal subset of C1 x C2
containing pairs of states (c1, c2) whose observable
unfoldings match at all depths (halting simultaneously
with identical labels, or branching simultaneously into
synchronized pairs).

## 4. Binary Coproducts

### 4.1 Algebras

#### 4.1.1 General Case

U: P-**Alg** -> **Set** does not create colimits. The
coproduct of (S1, alpha1) and (S2, alpha2) is constructed
by taking the disjoint union S1 + S2, freely generating
new syntax trees via the free monad, and taking the
quotient by the evaluation relations alpha1 and alpha2.

#### 4.1.2 Example: P(X) = A + X^2

The coproduct algebra contains evaluated elements from S1
and S2, alongside irreducible, formally constructed nodes
uniting elements from opposing sets (e.g.,
node_stuck(s1, s2) where s1 in S1 and s2 in S2).

### 4.2 Coalgebras

#### 4.2.1 General Case

Because U: P-**Coalg** -> **Set** creates colimits, the
coproduct of coalgebras (C1, gamma1) and (C2, gamma2) is
the disjoint union C1 + C2. The transition structure
applies gamma1 to elements of C1 and gamma2 to elements
of C2.

#### 4.2.2 Example: P(X) = A + X^2

The state space is C1 + C2. Transitions are isolated; a
state originating in C1 will only fork into subsequent
states strictly contained within C1.

## 5. Equalizers

### 5.1 Algebras

#### 5.1.1 General Case

U: P-**Alg** -> **Set** creates limits. Given parallel
homomorphisms f, g: (S1, alpha1) => (S2, alpha2), the
equalizer is the subset E = {s in S1 | f(s) = g(s)},
equipped with the restriction of alpha1.

#### 5.1.2 Example: P(X) = A + X^2

E is the maximal sub-evaluator of S1 where translations
f and g agree. Because f and g are homomorphisms,
evaluating trees strictly composed of elements in E
guarantees a result residing in E.

### 5.2 Coalgebras

#### 5.2.1 General Case

U: P-**Coalg** -> **Set** creates equalizers because
polynomial functors on **Set** preserve connected limits.
Given parallel homomorphisms
f, g: (C1, gamma1) => (C2, gamma2), the equalizer is the
subset E = {c in C1 | f(c) = g(c)}, with transitions
inherited from gamma1.

#### 5.2.2 Example: P(X) = A + X^2

E is the forward-invariant subspace of states where
observers f and g report identical unfoldings. If state
c in E forks into left and right children, the
homomorphism property forces both children to also
reside in E.

## 6. Coequalizers

### 6.1 Algebras

#### 6.1.1 General Case

U: P-**Alg** -> **Set** does not create colimits. Given
f, g: (S1, alpha1) => (S2, alpha2), the coequalizer is
formed by taking the set-theoretic quotient S2 / ~,
where ~ is the smallest congruence relation containing
(f(s), g(s)) for all s in S1, closed under the operations
of P.

#### 6.1.2 Example: P(X) = A + X^2

The coequalizer merges elements in S2 identified by f
and g, but must recursively merge any compound trees
built from those identified elements. If x ~ y, then
node2(x, z) ~ node2(y, z) must be enforced to maintain
the algebra structure.

### 6.2 Coalgebras

#### 6.2.1 General Case

U: P-**Coalg** -> **Set** creates colimits. Given
f, g: (C1, gamma1) => (C2, gamma2), the coequalizer is
the set-theoretic quotient Q = C2 / ~, where ~ is the
equivalence relation generated by f(c) ~ g(c). The
transition structure descends naturally to Q.

#### 6.2.2 Example: P(X) = A + X^2

The state space of C2 is partitioned into equivalence
classes. If c ~ d, their transitions must map to identical
equivalence classes in Q. This holds because f and g map
identical states in C1 to c and d, ensuring their
subsequent unfoldings remain homologous under the
quotient.

## 7. Pullbacks

### 7.1 Algebras

#### 7.1.1 General Case

U: P-**Alg** -> **Set** creates limits. Given
f: (S1, alpha1) -> (S3, alpha3) and
g: (S2, alpha2) -> (S3, alpha3), the pullback is the
fiber product S1 x_S3 S2 =
{(x, y) in S1 x S2 | f(x) = g(y)}, with pointwise
operations.

#### 7.1.2 Example: P(X) = A + X^2

The pullback algebra consists of pairs (t1, t2) from S1
and S2 that evaluate to the identical semantic element
in S3. Combining valid pairs via node constructs a new
pair guaranteed to remain in the pullback.

### 7.2 Coalgebras

#### 7.2.1 General Case

U: P-**Coalg** -> **Set** creates pullbacks. Given
f: (C1, gamma1) -> (C3, gamma3) and
g: (C2, gamma2) -> (C3, gamma3), the pullback is the
fiber product C1 x_C3 C2, with pointwise transitions.

#### 7.2.2 Example: P(X) = A + X^2

The pullback coalgebra pairs concrete states (c1, c2)
simulating the identical abstract state in C3. If
(c1, c2) forks, the shared observable specification
in C3 forces their respective left and right children
to also occupy the pullback subset.

## 8. Pushouts

### 8.1 Algebras

#### 8.1.1 General Case

U: P-**Alg** -> **Set** does not create colimits. Given
f: (S1, alpha1) -> (S2, alpha2) and
g: (S1, alpha1) -> (S3, alpha3), the pushout is
constructed by taking the coproduct algebra of S2 and S3,
and taking the coequalizer of the induced maps from S1.

#### 8.1.2 Example: P(X) = A + X^2

The pushout requires building the space of freely mixed
syntax between S2 and S3, and then equating any paths
that originated from the shared subdomain S1.

### 8.2 Coalgebras

#### 8.2.1 General Case

U: P-**Coalg** -> **Set** creates colimits. Given
f: (C1, gamma1) -> (C2, gamma2) and
g: (C1, gamma1) -> (C3, gamma3), the pushout is the
set-theoretic pushout (the disjoint union of C2 and C3
glued along the image of C1), with transitions applying
locally based on origin.

#### 8.2.2 Example: P(X) = A + X^2

The pushout state space amalgamates C2 and C3, treating
any state accessible via C1 as a shared bridge. Because
transitions are deterministic and preserved by
homomorphisms, a transition from a shared bridge state
remains unambiguously defined in the glued space.

## 9. Topos Structure (Coalgebras Only)

P-**Alg** generally lacks the internal logic or Cartesian
closure necessary to form a topos. P-**Coalg** constitutes
a Grothendieck topos, admitting both a subobject classifier
and exponential objects.

### 9.1 Subobject Classifier (Omega)

#### 9.1.1 General Case

The subobject classifier Omega in P-**Coalg** is a
coalgebra characterizing forward-closed subsets of states
(subcoalgebras).

#### 9.1.2 Example: P(X) = A + X^2

The states of Omega are finite and infinite binary trees
modeled on Z, but with every node and leaf assigned a truth
value from {True, False}. It enforces a strict transitional
invariant: if any node evaluates to True, all descendant
nodes must unconditionally evaluate to True.

### 9.2 Exponentials (Y^X)

#### 9.2.1 General Case

The exponential object Y^X in P-**Coalg** is a coalgebra
whose states represent dynamic mappings acting between the
state spaces of X and Y.

#### 9.2.2 Example: P(X) = A + X^2

A state f in Y^X is an evolving transition strategy mapping
states of X to states of Y. As machine X transitions and
forks into sub-states, the functional state f concurrently
transitions into an ordered pair of updated internal
strategies (f_left, f_right), structurally calibrated to
process the divergent future unfoldings of X.
