# Universal Property of ParametricFunctor

## Context

`ParametricFunctor E := RelSpanObj ‚•§ E` is the
category of functors from the relational span
category to an arbitrary target category `E`.
The copresheaf specialization
`ParametricCopresheaf := ParametricFunctor (Type 1)`
admits fully faithful embeddings of:

- Covariant endofunctors `(Type ‚•§ Type)` via the
  Barr extension (relation lifting)
- Contravariant functors `(Type·µí·µñ ‚•§ Type)` via
  pullback (contravariant relation lifting)
- Type expressions `TypeExprCat` via the full
  relational interpretation

All three embeddings share a structural pattern:
typeNode components carry the diagonal values
`F(I)` (or `F(op I)`, or `T.interp I I`), and
relNode components carry a subtype of the product
of two diagonal values, witnessing a relational
compatibility condition.

Similarly,
`PshParametricFunctor C E := PshRelSpanObj C ‚•§ E`
generalizes the source diagram to presheaves on an
arbitrary category `C`, with copresheaf
specialization
`PshParametricPresheaf C D := PshParametricFunctor C (D·µí·µñ ‚•§ Type w')`.
The equivalence `parametricFunctorEquiv E` witnesses
that `ParametricFunctor E ‚âå PshParametricFunctor (Discrete PUnit) E`
for any target category `E`.

## The question

Does `ParametricFunctor E` (or its copresheaf
specialization `ParametricCopresheaf`) have a
universal property within `Cat`?

### Motivation

Standard examples of copresheaf categories with
universal properties:

- `PSh(C) = C·µí·µñ ‚•§ Set` is the free cocompletion
  of `C`: for every cocomplete category `D`, the
  restriction functor `(PSh(C) ‚•§_cc D) ‚âÉ (C ‚•§ D)`
  is an equivalence (where `‚•§_cc` denotes
  cocontinuous functors).
- `Ind(C)` (ind-completion) is the free completion
  under filtered colimits.
- The Karoubi envelope is the free completion under
  idempotent splitting.

Each of these has a universal property of the form
"free completion under some class of colimits".

### What ParametricCopresheaf "does"

`ParametricCopresheaf` takes both covariant and
contravariant functors and embeds them into a
single category where:

1. Both variances appear on the same footing (as
   copresheaves on the same diagram category).
2. Only diagonal information is retained -- the
   embedding of a profunctor loses off-diagonal
   data.
3. Relational structure (at relNodes) mediates
   between elements at different types, replacing
   functorial transport (which is absent since
   `RelSpanObj` has no morphisms between
   typeNodes).

The general `ParametricFunctor E` retains this
diagram structure but the target category `E` need
not be a copresheaf category.

### Possible formulations

#### As a "diagonal completion"

Given the category `Type` with its self-enrichment,
one can form the diagonal functor
`Œî : Type ‚Üí Type √ó Type·µí·µñ` sending `I` to
`(I, I)`.  `ParametricCopresheaf` might be the
universal category through which both
`(Type ‚•§ Type)` and `(Type·µí·µñ ‚•§ Type)` factor via
their diagonals:

```text
(Type ‚•§ Type) ‚îÄ‚îÄ‚Üí ParametricCopresheaf ‚Üê‚îÄ‚îÄ (Type·µí·µñ ‚•§ Type)
      \                    ‚Üë                      /
       \                   |                     /
        \‚îÄ‚îÄ‚Üí (Type·µí·µñ √ó Type ‚•§ Type) ‚îÄ‚îÄ‚Üí‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ/
                    diag restriction
```

The profunctor category maps to
`ParametricCopresheaf` via diagonal restriction,
but this functor is neither full nor faithful.
`ParametricCopresheaf` might be the "universal
target for diagonal restriction" in some precise
sense.

#### As a "relational cocompletion"

`RelSpanObj` is generated by types and relations.
`ParametricCopresheaf = PSh(RelSpanObj)` (copresheaves
on `RelSpanObj`) might have a universal property
related to freely adjoining relational structure
to `Type`.  In this view, the covariant embedding
witnesses that relation lifting (Barr extension)
provides the "right" relational structure for
functors, and the contravariant embedding witnesses
the same for contravariant relation lifting.

#### As a "parametricity classifier"

By analogy with subobject classifiers in toposes:
a parametric functor classifies "parametricity
conditions".  The fully faithful embedding of
`TypeExprCat` shows that type expressions (the
standard syntactic source of parametricity
theorems) embed as a full subcategory.
`ParametricCopresheaf` might be the "semantic
completion" of `TypeExprCat` -- the category of
all possible parametricity conditions, including
those not expressible by finite type expressions.

### Related questions

1. `RelSpanObj` is the collage of
   `relSpanProfunctor`.  Does the copresheaf
   category of a collage have a known universal
   property relating it to the copresheaf
   categories of its components?

2. The twisted-arrow copresheaf topos
   `TwistedArrow(Type) ‚•§ Type` hosts paranatural
   transformations.  `ParametricCopresheaf` hosts
   parametric families.  Is there a span or cospan
   relating these two copresheaf categories, with
   the profunctor category (under either natural or
   paranatural morphisms) at the apex?

3. For the `endoProfCategory` (profunctors with
   paranatural morphisms): this category also
   operates on diagonal elements.  Is there a fully
   faithful functor
   `endoProfCategory ‚Üí ParametricCopresheaf`?
   The paranaturality condition (`DiagCompat`
   preservation) is structurally similar to the
   `profRelLift` condition at graph relations.

4. The equivalence `parametricFunctorEquiv E`
   generalizes from `ParametricFunctor E` to
   `PshParametricFunctor (Discrete PUnit) E` for
   any target category `E`.  Do the universal
   property questions for `ParametricFunctor`
   transfer to `PshParametricFunctor C` for
   arbitrary `C` via this equivalence and the
   `PshRelSpanObj` generalization?  The generalized
   `PshRelSpanObj C` uses `PshRel` (relations
   internal to presheaves) instead of `Prop`-valued
   relations.
