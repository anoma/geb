# Algebras and Coalgebras for Polynomial Functors

Let **Set** denote the category of sets and functions.
Let P: **Set** -> **Set** be an accessible polynomial
endofunctor. We examine the category of P-algebras,
denoted P-**Alg**, and the category of P-coalgebras,
denoted P-**Coalg**.

The running example is the functor P(X) = A + X^2,
representing binary trees with leaves labeled by elements
of a fixed set A.

## 1. Initial and Terminal Objects

### 1.1 Algebras

#### 1.1.1 General Case

The forgetful functor U: P-**Alg** -> **Set** strictly
creates limits. Consequently, the terminal algebra is
constructed upon the terminal object in **Set** (the
singleton set 1 = {*}), equipped with the unique structure
map P(1) -> 1.

Because P is accessible, P-**Alg** is cocomplete and
therefore possesses an initial object. The initial algebra
(W, alpha: P(W) ~= W) represents the type of strictly
well-founded trees (W-types) generated by the signature
of P. By Lambek's Lemma, alpha is an isomorphism.

#### 1.1.2 Example: P(X) = A + X^2

- **Terminal Algebra:** The underlying set is {*}. The
  operations are defined trivially: `leaf(a) = *` for all
  a in A, and `node(*, *) = *`.
- **Initial Algebra:** W is the set of all strictly finite
  binary trees with leaves labeled by A. The isomorphism
  W ~= A + W^2 dictates that every tree in W is uniquely
  either a distinct leaf a in A or a node joining two
  finite trees from W.

### 1.2 Coalgebras

#### 1.2.1 General Case

The forgetful functor U: P-**Coalg** -> **Set** strictly
creates colimits. Therefore, the initial coalgebra is
constructed upon the initial object in **Set** (the empty
set {}), equipped with the unique structure map {} -> P({}).

Because P is accessible, P-**Coalg** has a terminal
object. The terminal coalgebra (Z, zeta: Z ~= P(Z))
represents the type of all possible observable unfolding
behaviors (M-types). By Lambek's dual lemma, zeta is an
isomorphism.

#### 1.2.2 Example: P(X) = A + X^2

- **Initial Coalgebra:** The underlying set is {}. It
  represents a degenerate state machine with an empty
  state space.
- **Terminal Coalgebra:** Z is the set of all finite and
  infinite binary trees with leaves labeled by A. It
  constitutes the maximal state space of all possible
  valid binary unfoldings over A.

## 2. Free Monads and Cofree Comonads

### 2.1 Algebras (The Free Monad)

#### 2.1.1 General Case

Because P is accessible, the forgetful functor
U: P-**Alg** -> **Set** has a left adjoint
F: **Set** -> P-**Alg**. For any set Y, F(Y) is the free
P-algebra generated by Y. The composition U . F forms the
free monad M on **Set**. F(Y) is constructed as the
initial algebra for the modified functor P_Y(X) = Y + P(X).

#### 2.1.2 Example: P(X) = A + X^2

The free algebra on a set Y is the set of all finite
binary trees where the leaves are labeled by elements of
the disjoint union A + Y. Elements of Y act as freely
inserted, uninterpreted variables within the syntactic
tree structure.

### 2.2 Coalgebras (The Cofree Comonad)

#### 2.2.1 General Case

Because P is accessible, the forgetful functor
U: P-**Coalg** -> **Set** has a right adjoint
G: **Set** -> P-**Coalg**. For any set Y, G(Y) is the
cofree P-coalgebra over Y. The composition U . G forms
the cofree comonad C on **Set**. G(Y) is constructed as
the terminal coalgebra for the modified functor
P_Y(X) = Y * P(X).

#### 2.2.2 Example: P(X) = A + X^2

The cofree coalgebra on a set Y consists of the finite
and infinite binary trees from Z, but with a structural
addition: every valid state observation (every node and
every leaf) is explicitly annotated with an element of Y.
It represents the maximal unfolding behavior of P,
coupled with a continuous stream of outputs in Y at every
observational step.

## 3. Binary Products

### 3.1 Algebras

#### 3.1.1 General Case

Because U: P-**Alg** -> **Set** creates limits, the
binary product of algebras (S_1, alpha_1) and
(S_2, alpha_2) is the Cartesian product S_1 * S_2. The
structure map operates component-wise.

#### 3.1.2 Example: P(X) = A + X^2

The product algebra has the underlying set S_1 * S_2.
Evaluation occurs in parallel:

- leaf(a) = (leaf_1(a), leaf_2(a))
- node((x_1, x_2), (y_1, y_2)) =
  (node_1(x_1, y_1), node_2(x_2, y_2))

### 3.2 Coalgebras

#### 3.2.1 General Case

U: P-**Coalg** -> **Set** does not create products. The
binary product of coalgebras (C_1, gamma_1) and
(C_2, gamma_2) is the maximal synchronized bisimulation
between the two state spaces. It is constructed
equivalently as the pullback of C_1 and C_2 over the
terminal coalgebra Z in **Set**.

#### 3.2.2 Example: P(X) = A + X^2

The product coalgebra is the maximal subset of C_1 * C_2
containing pairs of states (c_1, c_2) whose observable
unfoldings match at all depths (halting simultaneously
with identical labels, or branching simultaneously into
synchronized pairs).

## 4. Binary Coproducts

### 4.1 Algebras

#### 4.1.1 General Case

U: P-**Alg** -> **Set** does not create colimits. The
coproduct of (S_1, alpha_1) and (S_2, alpha_2) is
constructed by taking the disjoint union S_1 + S_2,
freely generating new syntax trees via the free monad,
and taking the quotient by the evaluation relations
alpha_1 and alpha_2.

#### 4.1.2 Example: P(X) = A + X^2

The coproduct algebra contains evaluated elements from
S_1 and S_2, alongside irreducible, formally constructed
nodes uniting elements from opposing sets (e.g.,
node_stuck(s_1, s_2) where s_1 in S_1 and s_2 in S_2).

### 4.2 Coalgebras

#### 4.2.1 General Case

Because U: P-**Coalg** -> **Set** creates colimits, the
coproduct of coalgebras (C_1, gamma_1) and
(C_2, gamma_2) is the disjoint union C_1 + C_2. The
transition structure applies gamma_1 to elements of C_1
and gamma_2 to elements of C_2.

#### 4.2.2 Example: P(X) = A + X^2

The state space is C_1 + C_2. Transitions are isolated;
a state originating in C_1 will only fork into subsequent
states strictly contained within C_1.

## 5. Equalizers

### 5.1 Algebras

#### 5.1.1 General Case

U: P-**Alg** -> **Set** creates limits. Given parallel
homomorphisms f, g: (S_1, alpha_1) => (S_2, alpha_2),
the equalizer is the subset
E = {s in S_1 | f(s) = g(s)}, equipped with the
restriction of alpha_1.

#### 5.1.2 Example: P(X) = A + X^2

E is the maximal sub-evaluator of S_1 where translations
f and g agree. Because f and g are homomorphisms,
evaluating trees strictly composed of elements in E
guarantees a result residing in E.

### 5.2 Coalgebras

#### 5.2.1 General Case

U: P-**Coalg** -> **Set** creates equalizers because
polynomial functors on **Set** preserve connected limits.
Given parallel homomorphisms
f, g: (C_1, gamma_1) => (C_2, gamma_2), the equalizer
is the subset E = {c in C_1 | f(c) = g(c)}, with
transitions inherited from gamma_1.

#### 5.2.2 Example: P(X) = A + X^2

E is the forward-invariant subspace of states where
observers f and g report identical unfoldings. If state
c in E forks into left and right children, the
homomorphism property forces both children to also
reside in E.

## 6. Coequalizers

### 6.1 Algebras

#### 6.1.1 General Case

U: P-**Alg** -> **Set** does not create colimits. Given
f, g: (S_1, alpha_1) => (S_2, alpha_2), the coequalizer
is formed by taking the set-theoretic quotient S_2 / ~,
where ~ is the smallest congruence relation containing
(f(s), g(s)) for all s in S_1, closed under the
operations of P.

#### 6.1.2 Example: P(X) = A + X^2

The coequalizer merges elements in S_2 identified by f
and g, but must recursively merge any compound trees
built from those identified elements. If x ~ y, then
node_2(x, z) ~ node_2(y, z) must be enforced to maintain
the algebra structure.

### 6.2 Coalgebras

#### 6.2.1 General Case

U: P-**Coalg** -> **Set** creates colimits. Given
f, g: (C_1, gamma_1) => (C_2, gamma_2), the coequalizer
is the set-theoretic quotient Q = C_2 / ~, where ~ is
the equivalence relation generated by f(c) ~ g(c). The
transition structure descends naturally to Q.

#### 6.2.2 Example: P(X) = A + X^2

The state space of C_2 is partitioned into equivalence
classes. If c ~ d, their transitions must map to
identical equivalence classes in Q. This holds because f
and g map identical states in C_1 to c and d, ensuring
their subsequent unfoldings remain homologous under the
quotient.

## 7. Pullbacks

### 7.1 Algebras

#### 7.1.1 General Case

U: P-**Alg** -> **Set** creates limits. Given
f: (S_1, alpha_1) -> (S_3, alpha_3) and
g: (S_2, alpha_2) -> (S_3, alpha_3), the pullback is
the fiber product
`S_1 *_{S_3} S_2 = {(x, y) in S_1 * S_2 | f(x) = g(y)}`,
with pointwise operations.

#### 7.1.2 Example: P(X) = A + X^2

The pullback algebra consists of pairs (t_1, t_2) from
S_1 and S_2 that evaluate to the identical semantic
element in S_3. Combining valid pairs via node constructs
a new pair guaranteed to remain in the pullback.

### 7.2 Coalgebras

#### 7.2.1 General Case

U: P-**Coalg** -> **Set** creates pullbacks. Given
f: (C_1, gamma_1) -> (C_3, gamma_3) and
g: (C_2, gamma_2) -> (C_3, gamma_3), the pullback is
the fiber product C_1 *_{C_3} C_2, with pointwise
transitions.

#### 7.2.2 Example: P(X) = A + X^2

The pullback coalgebra pairs concrete states (c_1, c_2)
simulating the identical abstract state in C_3. If
(c_1, c_2) forks, the shared observable specification in
C_3 forces their respective left and right children to
also occupy the pullback subset.

## 8. Pushouts

### 8.1 Algebras

#### 8.1.1 General Case

U: P-**Alg** -> **Set** does not create colimits. Given
f: (S_1, alpha_1) -> (S_2, alpha_2) and
g: (S_1, alpha_1) -> (S_3, alpha_3), the pushout is
constructed by taking the coproduct algebra of S_2 and
S_3, and taking the coequalizer of the induced maps from
S_1.

#### 8.1.2 Example: P(X) = A + X^2

The pushout requires building the space of freely mixed
syntax between S_2 and S_3, and then equating any paths
that originated from the shared subdomain S_1.

### 8.2 Coalgebras

#### 8.2.1 General Case

U: P-**Coalg** -> **Set** creates colimits. Given
f: (C_1, gamma_1) -> (C_2, gamma_2) and
g: (C_1, gamma_1) -> (C_3, gamma_3), the pushout is the
set-theoretic pushout (the disjoint union of C_2 and C_3
glued along the image of C_1), with transitions applying
locally based on origin.

#### 8.2.2 Example: P(X) = A + X^2

The pushout state space amalgamates C_2 and C_3, treating
any state accessible via C_1 as a shared bridge. Because
transitions are deterministic and preserved by
homomorphisms, a transition from a shared bridge state
remains unambiguously well-defined in the glued space.

## 9. Topos Structure (Coalgebras Only)

P-**Alg** generally lacks the internal logic or Cartesian
closure necessary to form a topos. P-**Coalg** constitutes
a Grothendieck topos, admitting both a subobject classifier
and exponential objects.

### 9.1 Subobject Classifier (Omega)

#### 9.1.1 General Case

The subobject classifier Omega in P-**Coalg** is a
coalgebra characterizing forward-closed subsets of states
(subcoalgebras).

#### 9.1.2 Example: P(X) = A + X^2

The states of Omega are finite and infinite binary trees
modeled on Z, but with every node and leaf assigned a
truth value from {True, False}. It enforces a strict
transitional invariant: if any node evaluates to True, all
descendant nodes must unconditionally evaluate to True.

### 9.2 Exponentials (Y^X)

#### 9.2.1 General Case

The exponential object Y^X in P-**Coalg** is a coalgebra
whose states represent dynamic mappings acting between the
state spaces of X and Y.

#### 9.2.2 Example: P(X) = A + X^2

A state f in Y^X is an evolving transition strategy
mapping states of X to states of Y. As machine X
transitions and forks into sub-states, the functional
state f concurrently transitions into an ordered pair of
updated internal strategies (f_left, f_right),
structurally calibrated to process the divergent future
unfoldings of X.
